# Changelog - 2026-07-02

## Invoice Application Backend - Phase 1 Implementation

### Database Setup
*   Configured the application to connect to **Supabase PostgreSQL** via JDBC in `application.yaml`.
*   Integrated the Supabase Session Pooler connection URL.
*   Set Hibernate `ddl-auto: update` to automatically generate database schemas.

### Security & Authentication
*   Added `spring-boot-starter-security` and `jjwt` (JSON Web Token) dependencies to `build.gradle`.
*   Implemented **Stateless Session Management** using JWTs.
*   Created `SecurityConfig` to restrict all endpoints while allowing public access to `/api/auth/**`.
*   Implemented `JwtUtil` and `JwtFilter` to generate, parse, and validate JWTs from the `Authorization: Bearer <token>` header.
*   Integrated `BCryptPasswordEncoder` to securely hash and store passwords in the database.

### Entities & Data Layer
*   Created the primary `Company` entity which acts as the system user.
*   Implemented `UserDetails` interface in the `Company` entity for seamless Spring Security integration.
*   Fields added: `id` (UUID), `name`, `companyName`, `email`, `password`, `country`, `state`, `district`, `pincode`, `gst`, `address`, and `companyPhoneNo`.
*   Created `CompanyRepository` for Spring Data JPA database interactions.

### APIs
*   **`POST /api/auth/register`**: Registers a new company/user, hashes their password, and issues a JWT.
*   **`POST /api/auth/login`**: Authenticates users based on `email` and `password`, and issues a JWT.
*   Created corresponding DTOs (`RegisterRequest`, `LoginRequest`, `AuthResponse`) and a dedicated `AuthService` to handle the business logic.

---

### API Usage Examples (cURL)

**1. Register a new Company**
```bash
curl -X POST http://localhost:8080/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "name": "John Doe",
    "companyName": "Acme Corp",
    "email": "john@example.com",
    "password": "SecurePassword123",
    "country": "USA",
    "state": "California",
    "district": "San Francisco",
    "pincode": "94105",
    "gst": "123456789",
    "address": "123 Tech Lane",
    "companyPhoneNo": "+1234567890"
  }'
```

**2. Login as the Company**
```bash
curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "john@example.com",
    "password": "SecurePassword123"
  }'
```
