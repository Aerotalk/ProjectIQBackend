# Changelog: 2026-07-03 (Phase 2 - Enterprise Tenant Architecture)

## 🚀 What's New

### 1. The Organization (Root Tenant)
- **New Entity:** Created `Organization` to act as the absolute root of the hierarchy.
- **Role:** Represents the overarching corporate entity that owns everything else.

### 2. Company Refactor (Child of Organization)
- **Strict Relationship:** Refactored `Company` to belong to an `Organization` (`@ManyToOne`).
- **Normalized Schema:** Removed authentication and address fields from Company, isolating it to pure legal and tax information (GST, PAN, TAN, CIN, MSME, IEC).

### 3. Addresses and Banking
- **CompanyAddress:** A new entity attached to a `Company` supporting multiple locations (Registered vs. Corporate).
- **CompanyBankAccount:** A new entity attached to a `Company` storing highly specific financial details (IFSC, SWIFT, UPI).
- **Atomic API:** Refactored the `/api/admin/companies` endpoint to accept arrays of addresses and bank accounts in a single request, creating them all in a single transaction.

### 4. Enterprise RBAC
- **Organization Scoping:** Updated the `User` entity to belong to an `Organization` instead of a `Company`. Users now manage resources across the entire scope of their Organization.

### 5. True JPA Auditing & Soft Deletion
- **Automated Timestamps:** Enabled Spring Data JPA `@EnableJpaAuditing` to automatically populate `created_at` and `updated_at`.
- **Soft Delete:** Implemented `@SQLDelete` and `@SQLRestriction` on the core entities. Deleting them now safely populates the `deleted_at` column instead of destroying rows, providing a complete audit trail.

## 🛠️ Testing the Endpoints (cURL Commands)

### 1. Bootstrap the Super Admin
```bash
curl -X POST http://localhost:8080/api/auth/setup-super-admin \
  -H "Content-Type: application/json" \
  -d '{ 
        "name": "Super Admin", 
        "email": "admin@grivety.com", 
        "password": "SecurePassword123" 
      }'
```

### 2. Login as Super Admin
```bash
curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{ "email": "admin@grivety.com", "password": "SecurePassword123" }'
```
*(Copy the `token` from the response. You will need it for the next requests)*

### 3. Create an Organization
```bash
curl -X POST http://localhost:8080/api/admin/organizations \
  -H "Authorization: Bearer YOUR_SUPER_ADMIN_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{ 
        "organizationCode": "ORG001", 
        "organizationName": "Grivety Global",
        "legalName": "Grivety Global Pvt Ltd",
        "organizationType": "Pvt Ltd",
        "industry": "Technology",
        "status": "Active"
      }'
```
*(Copy the `id` of the organization from the response. You will need it for the next requests)*

### 4. Create a Company (with Address & Bank Account)

```bash
  -H "Authorization: Bearer YOUR_SUPER_ADMIN_JWT_TOKEN" \

        "organizationId": "THE_ORGANIZATION_UUID_FROM_ABOVE",
        "companyCode": "COMP001",
        "companyName": "Grivety India",
        "legalName": "Grivety India Software Pvt Ltd",

        "email": "hello@grivety.com",
        "phone": "+919876543210",
        "status": "Active",
        "addresses": [
          {
            "addressType": "Registered",
            "addressLine1": "123 Tech Park",
            "city": "Bengaluru",
            "state": "Karnataka",
            "country": "India",
```

### 5. Create an Organization Admin User
```bash
curl -X POST http://localhost:8080/api/admin/users \
  -H "Authorization: Bearer YOUR_SUPER_ADMIN_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{ 
        "name": "Jane Doe", 
        "email": "jane@grivety.com", 
        "password": "SecurePassword123",
        "organizationId": "THE_ORGANIZATION_UUID_FROM_ABOVE",
        "role": "ROLE_ORGANIZATION_ADMIN"
      }'
```

### 6. Login as the Organization Admin
```bash
curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
```
