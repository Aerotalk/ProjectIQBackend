# Changelog: 2026-07-03 (Phase 1.5 - Advanced Authentication)

## 🚀 What's New

### 1. Token Refresh & Secure Logout
- **Shorter Access Tokens:** Reduced the JWT Access Token lifespan to **15 minutes** for maximum security.
- **Refresh Tokens:** Created a new `RefreshToken` entity to store long-lived (7-day) tokens in the database.
- **Refresh Endpoint:** Added `POST /api/auth/refresh` to instantly exchange an expired access token for a new one without requiring the user to re-enter their password.
- **Logout Endpoint:** Added `POST /api/auth/logout` which actively deletes the refresh token from the database, permanently revoking access.

### 2. Account Verification & Password Resets
- **Email Verified Flag:** Added `emailVerified` (default `false`) to the `Company` entity.
- **Verification Tokens:** Created a `VerificationToken` entity to manage both `EMAIL_VERIFICATION` and `PASSWORD_RESET` tokens.
- **Testing Service:** Implemented a lightweight `EmailService` that acts as a development stub. Instead of requiring a 3rd party SMTP provider (like Mailtrap), it generates the verification links and logs them directly to the terminal for instant clickable testing.
- **New Endpoints:** Added `/api/auth/verify-email`, `/api/auth/forgot-password`, and `/api/auth/reset-password`.

## 🛠️ Testing the Endpoints (cURL Commands)

### 1. Register & Verify Email
```bash
# Register a new user
curl -X POST http://localhost:8080/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{ 
        "name": "Jane Doe", 
        "companyName": "Jane LLC", 
        "email": "jane@example.com", 
        "password": "SecurePassword123",
        "country": "India",
        "state": "Karnataka",
        "district": "Bengaluru",
        "pincode": "560001",
        "gst": "29ABCDE1234F1Z5",
        "address": "123 Tech Park, Whitefield",
        "companyPhoneNo": "+919876543210"
      }'

# Look at your terminal! You will see a link printed. Click it, or run this:
curl -X GET "http://localhost:8080/api/auth/verify-email?token=YOUR_VERIFICATION_TOKEN"
```

### 2. Login, Refresh, & Logout
```bash
# Login (Requires verified email)
curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{ "email": "jane@example.com", "password": "SecurePassword123" }'
# Note: Copy the "refreshToken" from the response.

# Refresh Token (Get a new 15-minute access token)
curl -X POST http://localhost:8080/api/auth/refresh \
  -H "Content-Type: application/json" \
  -d '{ "refreshToken": "YOUR_REFRESH_TOKEN" }'

# Logout (Permanently delete the refresh token from the DB)
curl -X POST http://localhost:8080/api/auth/logout \
  -H "Content-Type: application/json" \
  -d '{ "refreshToken": "YOUR_REFRESH_TOKEN" }'
```

### 3. Forgot & Reset Password
```bash
# Request a Password Reset
curl -X POST http://localhost:8080/api/auth/forgot-password \
  -H "Content-Type: application/json" \
  -d '{ "email": "jane@example.com" }'

# Look at your terminal! You will see a reset link printed. Copy the token.

# Reset Password
curl -X POST http://localhost:8080/api/auth/reset-password \
  -H "Content-Type: application/json" \
  -d '{ 
        "token": "YOUR_RESET_TOKEN", 
        "newPassword": "BrandNewPassword123!" 
      }'
```
